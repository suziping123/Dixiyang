# Dixiyang Vue 前端接口需求文档

## 接口需求概述

本文档详细描述了 Dixiyang Vue 前端应用所需的所有后端 API 接口，包括接口功能、请求参数和期望响应格式。

## 认证相关接口

### 1. 用户登录

**接口地址**: `POST /auth/login`

**功能描述**: 用户通过用户名和密码进行登录，获取 JWT Token 和用户信息

**请求参数**:
```typescript
{
  username: string,  // 用户名
  password: string   // 密码
}
```

**期望响应**:
```typescript
{
  code: 200,
  msg: "success",
  data: {
    token: string,   // JWT Token
    user: {
      id: number,
      username: string,
      nickname: string,
      email?: string
    }
  }
}
```

### 2. 用户注册

**接口地址**: `POST /auth/register`

**功能描述**: 新用户注册账户

**请求参数**:
```typescript
{
  username: string,   // 用户名
  password: string,   // 密码
  nickname: string,   // 昵称
  email?: string      // 邮箱（可选）
}
```

**期望响应**:
```typescript
{
  code: 200,
  msg: "注册成功",
  data: null
}
```

## 小说管理接口

### 1. 获取小说列表

**接口地址**: `GET /novel/listall?page=1&pageSize=10`

**功能描述**: 分页获取当前用户的小说列表，包含统计信息

**请求参数**:
- `page`: 页码（默认 1）
- `pageSize`: 每页大小（默认 10）

**期望响应**:
```typescript
{
  code: 200,
  msg: "获取成功",
  data: {
    records: Array<{
      id: number,
      title: string,
      penName: string,
      description: string,
      coverUrl: string,
      userId: number,
      createTime: string,
      updateTime: string,
      charCount: number,    // 角色数量
      nodeCount: number,    // 节点数量
      relationCount: number // 关系数量
    }>,
    total: number,
    size: number,
    current: number,
    pages: number
  }
}
```

### 2. 创建小说

**接口地址**: `POST /novel/create`

**功能描述**: 创建新小说

**请求参数**:

**接口地址**: `POST /chat`

## 接口一致性要求

1. **统一响应格式**: 所有接口使用相同的响应结构 `{code, msg, data}`
2. **错误处理**: 失败时返回适当的错误码和错误信息
3. **认证机制**: 需要认证的接口应在请求头中携带 JWT Token
4. **分页规范**: 分页参数从 1 开始，返回包含总记录数和页数信息
5. **时间格式**: 统一使用 ISO 8601 标准格式
