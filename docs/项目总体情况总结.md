# Dixiyang 项目总体情况总结

## 项目概述

Dixiyang 是一个小说创作管理平台，采用前后端分离架构，包含后端服务（dixiyang-engine）和前端应用（dixiyang-vue）。该项目旨在为小说创作者提供全方位的工具支持，包括小说管理、角色管理、故事节点编辑、时间线管理以及基于AI的RAG智能创作助手等功能。

## 技术栈

### 后端 (dixiyang-engine)
- **框架**: Spring Boot 3.4.2
- **ORM**: MyBatis Plus 3.5.10.1
- **安全**: Spring Security + JWT
- **数据库**: MySQL 8.0+
- **向量数据库**: Qdrant (用于RAG系统)
- **缓存**: Redis
- **AI集成**: Spring AI 1.0.0-M5
- **代码生成**: MyBatis Plus Generator

### 前端 (dixiyang-vue)
- **框架**: Vue 3.5.29
- **构建工具**: Vite 7.3.1
- **状态管理**: Pinia 3.0.4
- **路由**: Vue Router 5.0.3
- **UI组件**: Element Plus 2.13.5
- **HTTP客户端**: Axios 1.13.6
- **动画库**: GSAP 3.14.2, Motion 12.38.0
- **样式**: Tailwind CSS 4.2.1

## 核心功能模块

### 1. 用户认证与授权
- 用户注册/登录
- JWT令牌认证
- 权限控制

### 2. 小说管理
- 创建、编辑、删除小说
- 小说列表展示（分页）
- 小说详情查看
- 封面上传

### 3. 角色管理
- 角色的CRUD操作
- 按小说分类管理角色
- 角色属性管理（姓名、性别、年龄、外貌、性格、背景等）

### 4. 故事节点管理
- 故事节点的创建和管理
- 节点内容编辑
- 节点间关系管理

### 5. 时间线管理
- 时间线的创建和管理
- 时间线与节点关联

### 6. RAG智能创作助手
- 基于向量数据库的检索增强生成
- 根据选定角色和节点生成创作建议
- AI对话功能

### 7. 文件管理
- 文件上传和存储
- 封面图片管理

### 8. 用户配置
- 个性化设置管理
- 界面主题定制
- 创作偏好设置

## 数据库结构

主要数据表包括：
- `app_user`: 用户信息表
- `novels`: 小说信息表
- `novel_character`: 角色信息表
- `story_node`: 故事节点表
- `timeline`: 时间线表
- `novel_relation`: 小说关系表
- `file`: 文件信息表
- `user_config`: 用户配置表

## 项目架构特点

1. **分层架构**: Controller -> Service -> Mapper -> Database
2. **统一响应格式**: 使用Result类封装API响应
3. **JWT认证**: 无状态的身份验证机制
4. **RESTful API**: 遵循REST设计原则
5. **RAG系统集成**: 结合向量数据库实现智能搜索和推荐
6. **模块化设计**: 各功能模块职责清晰

## 当前开发状态

- 基础功能已完成：用户认证、小说管理、角色管理
- 部分功能待完善：故事节点、时间线管理的前端集成
- RAG系统已实现基本功能，但需要进一步优化
- 文件上传功能需要完善
- 用户配置功能正在开发中

## 存在的问题和改进方向

1. **接口一致性**: 前后端接口定义需要进一步统一
2. **错误处理**: 需要更完善的异常处理机制
3. **性能优化**: 大数据量下的查询性能需要优化
4. **安全性**: 需要加强输入验证和SQL注入防护
5. **测试覆盖**: 需要增加单元测试和集成测试

## 部署建议

1. 后端使用Docker容器化部署
2. 前端通过CDN分发静态资源
3. 数据库主从复制提高可用性
4. Redis集群提升缓存性能
5. Nginx反向代理和负载均衡
