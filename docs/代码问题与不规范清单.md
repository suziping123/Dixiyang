#### 2. ⚠️ 使用 System.out.println 代替日志框架

**问题位置**:
- `Result.java` (L20, L29)
- `AuthServiceImpl.java` (L48, L49)
- `ChatController.java` (L107, L116)

**问题代码**:
```java
// Result.java
public static <T> Result<T> success(T data) {
    System.out.println("success");  // ❌ 不应该在生产代码中使用
    return new Result<T>(200, "操作成功", data);
}

// AuthServiceImpl.java
if (!passwordEncoder.matches(password, appUser.getPassword())) {
    System.out.println(password);  // ❌ 严重安全问题！打印明文密码
    System.out.println(passwordEncoder.encode(password));
    return Result.error("用户名或密码错误");
}
```

**问题**:
1. **安全风险**: 打印明文密码到控制台
2. **性能问题**: System.out 是同步操作，影响性能
3. **无法控制**: 不能按级别过滤（INFO/WARN/ERROR）
4. **难以维护**: 生产环境无法关闭输出

**正确做法**:
```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

@Service
public class AuthServiceImpl implements AuthService {
    private static final Logger log = LoggerFactory.getLogger(AuthServiceImpl.class);
    
    @Override
    public Result login(String username, String password) {
        AppUser appUser = appUserMapper.selectOne(queryWrapper);
        if (appUser == null) {
            log.warn("登录失败：用户不存在, username={}", username);  // ✅ 使用日志
            return Result.error("用户名或密码错误");
        }
        
        if (!passwordEncoder.matches(password, appUser.getPassword())) {
            log.warn("登录失败：密码错误, username={}", username);  // ✅ 不打印密码
            return Result.error("用户名或密码错误");
        }
        // ...
    }
}
```

**工作量**: 1-2小时

#### 4. ⚠️ getCurrentUserId() 方法吞没异常

**问题位置**: `UserServiceIml.java` L97-114

**问题代码**:
```java
private Long getCurrentUserId() {
    try {
        String token = request.getHeader("Authorization");
        if (token == null || !token.startsWith("Bearer ")) {
            throw new RuntimeException("未获取到有效token");
        }
        token = token.substring(7);
        String userIdStr = jwtUtils.getUserIdFromToken(token);
        if (userIdStr == null || userIdStr.isEmpty()) {
            throw new RuntimeException("token中未包含用户ID");
        }
        return Long.valueOf(userIdStr);
    } catch (Exception e) {
        // 开发环境默认返回1，生产环境需抛出异常
        return 1L;  // ❌ 吞没所有异常，返回硬编码值
    }
}
```

**问题**:
1. **安全隐患**: 任何异常都返回用户ID=1，可能导致越权访问
2. **调试困难**: 真正的错误被隐藏
3. **生产风险**: 注释说"生产环境需抛出异常"但代码未实现

**正确做法**:
```java
private Long getCurrentUserId() {
    try {
        String token = request.getHeader("Authorization");
        if (token == null || !token.startsWith("Bearer ")) {
            throw new AuthenticationException("未获取到有效token");
        }
        token = token.substring(7);
        String userIdStr = jwtUtils.getUserIdFromToken(token);
        if (userIdStr == null || userIdStr.isEmpty()) {
            throw new AuthenticationException("token中未包含用户ID");
        }
        return Long.valueOf(userIdStr);
    } catch (AuthenticationException e) {
        log.error("获取用户ID失败: {}", e.getMessage());
        throw e;  // ✅ 抛出认证异常
    } catch (Exception e) {
        log.error("解析token异常", e);
        throw new AuthenticationException("无效的token", e);
    }
}
```

**工作量**: 30分钟

#### 5. ⚠️ Result.success() 返回类型不一致

**问题位置**: `Result.java` L19-26

**问题代码**:
```java
public static <T> Result<T> success(T data) {
    System.out.println("success");
    return new Result<T>(200, "操作成功", data);
}

// 支持自定义 msg
public static <T> Result<T> success(String msg, T data) {
    return new Result<T>(200, msg, data);
}
```

**问题**:
- 第一个方法固定返回 "操作成功"
- 第二个方法可以自定义消息
- 导致调用者困惑该用哪个

**现状**:
```java
// 有的地方用第一个
return Result.success(data);

// 有的地方用第二个
return Result.success("获取成功", novelPage);
```

**建议**: 统一使用带消息的版本，或者删除第一个方法

**工作量**: 1小时（需要修改所有调用处）

## 🟡 中优先级问题（建议修复）

### 后端问题

#### 9. ⚠️ 缺少全局异常处理

**问题**: 项目中没有 `@ControllerAdvice` 全局异常处理器

**影响**:
- 每个Controller都要自己处理异常
- 异常响应格式不统一
- 错误信息可能泄露堆栈跟踪

**建议添加**:
```java
@ControllerAdvice
@Slf4j
public class GlobalExceptionHandler {
    
    @ExceptionHandler(AuthenticationException.class)
    public Result<Void> handleAuthException(AuthenticationException e) {
        log.warn("认证失败: {}", e.getMessage());
        return Result.error("认证失败: " + e.getMessage());
    }
    
    @ExceptionHandler(RuntimeException.class)
    public Result<Void> handleRuntimeException(RuntimeException e) {
        log.error("运行时异常", e);
        return Result.error("服务器内部错误");
    }
    
    @ExceptionHandler(Exception.class)
    public Result<Void> handleException(Exception e) {
        log.error("系统异常", e);
        return Result.error("系统繁忙，请稍后重试");
    }
}
```

**工作量**: 2小时

**示例**:

```java

    // ...
```

#### 13. ⚠️ 重复的 API 调用逻辑

**问题**: 多个组件中都有相似的小说列表获取逻辑

**示例**:
- `HomeView.vue` 中有 `fetchNovels()`
- `RagAssistantView.vue` 中也有获取小说列表的代码

**建议**: 提取到 composable 或 store 中

```typescript
// composables/useNovels.ts
export function useNovels() {
  const novels = ref<Novel[]>([])
  const isLoading = ref(false)
  
  const fetchNovels = async (page = 1, pageSize = 10) => {
    isLoading.value = true
    try {
      const res = await getNovelList(page, pageSize)
      novels.value = res.data.records
    } finally {
      isLoading.value = false
    }
  }
  
  return { novels, isLoading, fetchNovels }
}
```
