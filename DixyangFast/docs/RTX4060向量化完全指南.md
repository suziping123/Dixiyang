## 1. 硬件自动检测

**文件**：`rag-shared/python/rag_shared/config.py:79-109`

GPU 检测逻辑由 `detect_hardware()` 完成，决定使用哪套 YAML 配置：

```python
def detect_hardware() -> str:
    if torch.cuda.is_available():
        gpu_name = torch.cuda.get_device_name(0).lower()
        vram_gb = torch.cuda.get_device_properties(0).total_memory / 1024**3

        logger.info(f"检测到 GPU: {gpu_name}, VRAM: {vram_gb:.1f}GB")

        if "4060" in gpu_name or "rtx 4060" in gpu_name:
            return "rtx_4060"        # ← 精确匹配 RTX 4060
        elif vram_gb >= 6:
            return "rtx_4060"        # ← 其他 6GB+ GPU 复用 4060 配置
        else:
            logger.warning(f"GPU 显存较小 ({vram_gb:.1f}GB)，回退到 CPU 模式")
            return "cpu_fallback"

    cpu_info = platform.processor().lower()
    if "5600u" in cpu_info or "ryzen 5 5600u" in cpu_info:
        return "r5_5600u"

    return "cpu_fallback"
```

> 返回的字符串与 `HARDWARE_PRESETS` 映射表匹配，决定加载哪个 YAML 文件：
> ```python

>     "r5_5600u": "rag_r5_5600u.yaml",
>     "rtx_4060": "rag_rtx_4060.yaml",
>     "cpu_fallback": "rag_r5_5600u.yaml",
> }
> ```

```python
def load_config(hardware=None, config_dir=None, base_config_name="rag_base.yaml"):
    # 1. 加载基础配置
    base = load_yaml(config_dir / base_config_name)

    # 2. 自动检测硬件（若未指定）
    if hardware is None or hardware == "auto":
        hardware = detect_hardware()

    # 3. 加载硬件配置
    preset_file = HARDWARE_PRESETS.get(hardware)
    hw_config = load_yaml(config_dir / preset_file)

    # 4. 深度合并（硬件覆盖基础）
    merged = deep_merge(base, hw_config)

    # 5. 解析为 RAGConfig 对象
    cfg = parse_config(merged)

    # 6. 环境变量覆盖（最高优先级）
    _apply_env_overrides(cfg)

    return cfg
```

## 3. 加载 bge-m3 模型与 FP16 加速

### 3.1 EmbeddingModel 类（离线构建用）

**文件**：`rag-shared/python/rag_shared/embeddings.py`（66 行）

核心离线构建时的 Embedding 封装，加载 `sentence-transformers` 的 `bge-m3` 模型：

```python
class EmbeddingModel:
    def __init__(self, config: RAGConfig):
        self.config = config
        self.model = None
        self._load_model()

    def _load_model(self):
        from sentence_transformers import SentenceTransformer

        self.model = SentenceTransformer(
            self.config.embedding_model,         # "BAAI/bge-m3"
            device=self.config.embedding_device,  # "cuda"
            cache_folder=str(cache_dir)
        )

        #FP16 半精度：显存占用减半，速度翻倍
        if self.config.embedding_half_precision and self.config.embedding_device == 'cuda':
            self.model.half()
            logger.info("启用 FP16 半精度")

    def encode(self, texts: List[str]) -> List[List[float]]:
        # 分批编码，防止 OOM
        MAX_ENCODE = 5000
        embeddings = self.model.encode(
            texts,
            batch_size=self.config.embedding_batch_size,  # GPU=64
            show_progress_bar=True,
            convert_to_numpy=True,
            normalize_embeddings=True                     # L2 归一化
        )
        return embeddings.tolist()
```

### 3.2 DixiyangEmbeddings 类（生产查询用）

**文件**：`DixyangFast/src/dixiyang/services/embeddings.py`（45 行）

生产环境下的 Embedding，**优先 HTTP 调用 Python Embedding 服务，降级本地 CPU 推理**：

```python
class DixiyangEmbeddings(Embeddings):
    def _get_embedding(self, text: str) -> list[float]:
        # 路径 1：调用远程 Embedding 服务（支持 GPU）
        try:
            import httpx
            resp = httpx.post(
                "http://localhost:8085/api/rag/embed",   # Python RAG 服务
                json={"text": text},
                timeout=10,
            )
            if resp.status_code == 200:
                return resp.json().get("embedding")
        except Exception:
            pass

        # 路径 2：降级本地 CPU 模型
        try:
            if self._model is None:
                model_path = os.getenv(
                    "RAG_EMBEDDING_MODEL",
                    str(Path(__file__).parent.parent.parent.parent / "models" / "bge-m3"),
                )
                from sentence_transformers import SentenceTransformer
                self._model = SentenceTransformer(model_path, device="cpu")
            emb = self._model.encode([text], normalize_embeddings=True).tolist()[0]
            return emb
        except Exception as e:
            raise RuntimeError(f"Embedding 生成失败: {e}") from e
```

> **设计意图**：双路径确保高可用——查询时先尝试 GPU 服务，失败则降级 CPU，不中断业务。

### FP16 加速原理

```python
# sentence_transformers 模型默认 FP32（单精度，4 字节/参数）
# model.half() 将其转换为 FP16（半精度，2 字节/参数）
# RTX 4060 原生支持 FP16 计算（Tensor Cores），bge-m3 也官方支持

self.model = SentenceTransformer("BAAI/bge-m3", device="cuda")
print(next(self.model.parameters()).dtype)  # → torch.float32

self.model.half()
print(next(self.model.parameters()).dtype)  # → torch.float16
```

**加速效果**：

| 指标 | FP32 | FP16 | 提升 |
|------|------|------|------|
| 显存占用 | ~1200 MB | ~600 MB | **50% ↓** |
| 嵌入速度 | ~80 条/秒 | ~150 条/秒 | **87% ↑** |
| 精度损失 | — | <0.5% | 可忽略 |

**文件**：`rag-shared/python/rag_shared/loaders.py:120-127`

### 4.1 Markdown 标题层级切分 + 递归切分

```python
class MarkdownLoader(BaseLoader):
    def load(self, file_path: Path) -> Generator[DocumentChunk, None, None]:
        text = file_path.read_text(encoding='utf-8')

        # Step 1：按 `#` / `##` 标题切分出章节段落
        sections = self._split_by_headers(text)

        for section in sections:
            # Step 2：对每个章节用 RecursiveCharacterTextSplitter 再切
            sub_chunks = self._split_text(section['content'])
            for sub_chunk in sub_chunks:
                if len(sub_chunk.strip()) < 50:      # 过滤过短块
                    continue
                yield DocumentChunk(
                    content=sub_chunk.strip(),
                    metadata={
                        "source": "book",
                        "book_title": book_title,
                        "category": category,
                        "chapter": section.get('h1', ''),
                        "section": section.get('h2', ''),
                        "chunk_id": f"{book_title}_{index}",
                    },
                    ...
                )

    def _split_by_headers(self, text: str) -> List[Dict[str, str]]:
        """按 # 标题层级切分章节"""
        for line in lines:
            h1_match = re.match(r'^#\s+(.+)$', line)
            h2_match = re.match(r'^##\s+(.+)$', line)
            if h1_match:     # 新一级标题 → 新章节
                ...
            elif h2_match:   # 新二级标题 → 新子节
                ...

    def _split_text(self, text: str) -> List[str]:
        """语义边界递归切分"""
        splitter = RecursiveCharacterTextSplitter(
            chunk_size=600,       # GPU 配置
            chunk_overlap=120,    # 重叠 20%
            separators=[          # 优先级从高到低
                "\n\n",           # 段落边界
                "\n",             # 行边界
                "。", "！", "？",  # 句边界（中文）
                "；", "，", " ",  # 短语边界
                ""                # 字符边界（保底）
            ],
            keep_separator=True,
        )
        return splitter.split_text(text)
```

### 4.2 GPU vs CPU 切分配置

| 参数 | RTX 4060 (GPU) | R5 5600U (CPU) | 原因 |
|------|:---:|:---:|------|
| `chunk_size` | 600 | 600 | 同模型 |
| `chunk_overlap` | 120 | 120 | 同模型 |
| `separators` | 相同 | 相同 | 同模型 |

> `chunk_size=600` 与 `bge-m3` 的 8192 token 上下文窗口相比很小，这是为了检索精度——短 chunk 语义更集中。

## 5. ChromaDB 向量存储

### 5.1 VectorDB 封装

```python
class VectorDB:
    def _init_db(self):
        import chromadb
        from chromadb.config import Settings

        persist_dir = Path(self.config.vectordb_path)  # ./storage/vectordb_4060
        persist_dir.mkdir(parents=True, exist_ok=True)

        self.client = chromadb.PersistentClient(
            path=str(persist_dir),
            settings=Settings(anonymized_telemetry=False, allow_reset=True)
        )

        self.collection = self.client.get_or_create_collection(
            name=self.config.collection_name,          # "dixiyang_knowledge"
            metadata={"hnsw:space": "cosine"}          # ✓ 余弦距离
        )

    def add(self, chunks: List[DocumentChunk], embeddings: List[List[float]]):
        """批量写入（含原文+向量+元数据）"""
        self.collection.add(
            ids=[c.chunk_id for c in chunks],
            documents=[c.content for c in chunks],      # 原文
            embeddings=embeddings,                       # 1024d 向量
            metadatas=[c.metadata for c in chunks]       # 来源/标签
        )

    def count(self) -> int:
        return self.collection.count()
```

```python
def _rerank_results(query: str, documents: list[str]) -> list[int]:
    reranker_url = os.getenv("RERANKER_URL", "http://localhost:8085")
    try:
        import httpx
        resp = httpx.post(
            f"{reranker_url}/api/rerank",
            json={"query": query, "documents": documents},
            timeout=10,
        )
        if resp.status_code == 200:
            data = resp.json()
            results = data.get("results", [])
            doc_to_idx = {d: i for i, d in enumerate(documents)}
            return [doc_to_idx[r["content"]] for r in results if r["content"] in doc_to_idx]
    except Exception as e:
        log.warning("Reranker 不可用，使用向量检索原始排序: %s", e)
    return list(range(len(documents)))    # 降级：保持原序
```

```python
async def rerank(req: RerankRequest):
    pairs = [[req.query, doc] for doc in req.documents]
    scores = _reranker.compute_score(pairs)
    scored = sorted(zip(req.documents, scores), key=lambda x: x[1], reverse=True)
    results = [{"content": doc, "score": float(score)} for doc, score in scored]
    return {"results": results[:req.top_k]}
```

### 6.4 KnowledgeSearchTool 完整检索流程

**文件**：`DixyangFast/src/dixiyang/services/knowledge_search_tool.py:57-92`

```python
@tool
def knowledge_search(query: str, top_k: int = 5, doc_type: str | None = None) -> str:
    """搜索小说知识库（角色/世界观/大纲/参考资料）"""
    vectorstore = _get_vectorstore()

    # 1. 宽召回：取 topK×4 条候选
    recall_k = min(top_k * 4, 100)
    results = vectorstore.similarity_search_with_score(query, k=recall_k)

    # 2. 类型过滤
    candidates = []
    for doc, _score in results:
        if doc_type and not _matches_type(doc.metadata, doc_type):
            continue
        candidates.append({"doc": doc.page_content, "source": ...})

    # 3. Reranker 精排
    doc_texts = [c["doc"] for c in candidates]
    ranked = _rerank_results(query, doc_texts)
    reranked = [candidates[i] for i in ranked[:top_k]]

    return "\n\n".join([f"{i}. {c['doc'][:500]}（来源：{c['source']}）" for i, c in enumerate(reranked, 1)])
```

### 7.2 核心 run() 方法

```python
def run(self, incremental: Optional[bool] = None):
    # 1. 收集文件
    files = self.collect_files()

    # 2. 增量过滤（仅处理变更文件）
    if incremental:
        files_to_process = [f for f in files if self.state_tracker.is_changed(f)]
    else:
        files_to_process = files

    # 3. 逐个文件处理
    for file_path in tqdm(files_to_process):
        try:
            chunk_count = self.process_file(file_path)  # ↓ 见下方
            if chunk_count > 0:
                self.state_tracker.update(file_path, chunk_count)
        except Exception as e:
            ...
    self.state_tracker.save()
    self._save_stats()
    return self.stats
```

### 7.3 单文件处理（嵌入 + 写入）

```python
def process_file(self, file_path: Path) -> int:
    loader = self.get_loader(file_path)          # 自动选择 .md/.txt/.csv/.json
    chunks = list(loader.load(file_path))        # 加载 + 切分

    # 分批编码（每批 5000 条防 OOM）
    BATCH = 5000
    for start in range(0, len(chunks), BATCH):
        batch = chunks[start:start + BATCH]
        texts = [c.content for c in batch]
        embs = self.embedding_model.encode(texts)  # GPU 批量编码

        # 分批写入向量库（每批 5000）
        DB_BATCH = 5000
        for batch_start in range(0, len(ids), DB_BATCH):
            self.vectordb.collection.add(
                ids=ids[batch_start:batch_start + DB_BATCH],
                documents=docs[batch_start:batch_start + DB_BATCH],
                embeddings=embs[batch_start:batch_start + DB_BATCH],
                metadatas=metas[batch_start:batch_start + DB_BATCH],
            )

    self._backup_chunks(chunks, file_path)  # 备份 JSONL
    return len(chunks)
```

## 9. 双机协作

### 核心思想

```
Windows (RTX 4060, GPU)          Linux Server (CPU)
────────────────────────          ──────────────────
bge-m3 + FP16 + batch=64         ChromaDB Server（只查不建）
→ 构建 111509 条嵌入             Python RAG 服务（CPU 推理）
→ 存储到 vectordb_4060/          bge-m3 CPU + bge-reranker CPU
→ scp 到 Linux
```

### 前提

两台机器必须用**同一个 Embedding 模型**（`BAAI/bge-m3`，1024 维），否则向量空间不一致。

### 做法步骤

```bash
# ===== Windows (RTX 4060) =====
# 1. 全量构建向量库
cd rag-shared

# 2. 打包传输

scp vectordb_4060.tar.gz user@linux-server:/path/to/DixyangFast/storage/

# ===== Linux Server (CPU) =====
# 3. 解压到同路径
cd /path/to/DixyangFast
tar -xzf vectordb_4060.tar.gz -C storage/

# 4. 启动 RAG 服务
bash start_rag.sh

# 5. 验证

# → {"status":"ok","chromadb_connected":true,"total_documents":111509}
```

## 12. GPU 与 CPU 全参数对比

| 参数 | RTX 4060 (GPU) | R5 5600U (CPU) | 差异原因 |
|------|:---:|:---:|----------|
| `device` | `cuda` | `cpu` | 硬件差异 |
| `precision` | `fp16` | `fp32` | GPU 支持 FP16 Tensor Cores |
| `embedding.batch_size` | 64 | 32 | GPU 并行能力强 |
| `embedding.max_length` | 8192 | 512 | 长文本可省去预切分 |
| `embedding.half_precision` | true | false | CPU 无 FP16 加速指令 |
| `text_splitter.chunk_size` | 600 | 600 | 相同模型 |
| `retrieval.top_k` | 8 | 5 | GPU 可处理更多候选 |
| `retrieval.use_reranker` | true | false | Reranker GPU 推理才能实时 |
| `reranker_model` | `bge-reranker-v2-m3` | — | CPU 不支持 |
| `processing.num_workers` | 2 | 10 | GPU I/O 等待少 |
| `processing.batch_size` | 128 | 256 | CPU 更大更优 |
| `vectordb.path` | `./storage/vectordb_4060` | （同左） | **双机共享同一路径** |
| `memory.reserved_vram_mb` | 1024 | — | 预留显存 |

> **后向兼容**：CPU 配置的 `vectordb.persist_directory` 与 GPU 相同，Windows 构建后复制到 Linux 即可直接查询。
